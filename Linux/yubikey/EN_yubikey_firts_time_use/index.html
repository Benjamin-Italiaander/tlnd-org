YubiKey bare minimum Link to heading

A short Getting Started Using the Linux CLI Link to heading

Step 1 till step 8 are the absolute bare minimum you should complete before you start using your YubiKey. If you skip these steps, there is nothing truly secure about using your YubiKey.

Hardware tokens are a powerful tool and can raise your security to a very high level—but only when used correctly. Be aware that using hardware tokens incorrectly will not result in a more secure user environment.

This instruction is a modified document based on the YubiKey guide found here Securing SSH with FIDO2

Never use SSH agent forwarding with your YubiKey — or use of ssh -a — in fact, it’s best not to use agent forwarding at all.

  1. Install the Necessary Software

There are more ways of installing my advice is to follow this instruction, but in case you are in a rush or a less advanced user you can use the apt repo, in my case (debian based) I added the needed packages:

sudo apt-add-repository ppa:yubico/stable
sudo apt update
sudo apt install yubikey-manager yubikey-personalization-gui
sudo apt install yubikey-personalization-gui sudo libpam-yubico
sudo apt install libpam-yubico
sudo apt install yubikey-manager-qt
  1. Verify the YubiKey

Run the ykman info command and check if your device is detected:

ykman info

  1. FIDO2

Change Default FIDO2 PIN Link to heading

For stronger security, set a FIDO2 PIN on your YubiKey:

ykman fido access change-pin

You will be prompted for:

  • current PIN (or empty if default)

  • new PIN

  • confirmation

  1. Verify the PIN
ykman fido access verify-pin
Enter your PIN:
PIN verified.

PIV (Smartcard) Link to heading

Default Values Link to heading

To change PIV access, you need the default credentials:

  • Default PIN: 123456

  • Default PUK: 12345678

  • Default management key: (Yubico default 24-byte key unless changed)

  1. Change PIN
ykman piv access change-pin
  1. Change PUK
ykman piv access change-puk
  1. Change the Management Key

I recommend generating a new management key automatically:

ykman piv access change-management-key --protect --generate

SAVE your management key and PUK somewhere safe.
I made a QR code from it and stored it offline. I have a instruction here about how to generate a qr-code in you cli

Now check if the PIN works — this command should ask for it:

ykman fido credentials list
